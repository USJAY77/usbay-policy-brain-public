# PB-017 Governance Action Tracking

## Purpose

PB-017 tracks execution and completion status of governance improvement actions generated by PB-016.

The control reads the PB-016 improvement plan, roadmap, and action register. It then generates local tracking, progress, completion, and dashboard reports.

## Scope

PB-017 evaluates:

- open actions
- completed actions
- overdue actions
- blocked actions
- governance progress percentage
- roadmap completion percentage

PB-017 generates:

- `pb017_governance_action_tracker.json`
- `pb017_governance_progress_report.json`
- `pb017_governance_completion_report.json`
- `pb017_governance_status_dashboard.json`

## Authority Boundary

PB-017 is local governance tracking only.

PB-017 must not call:

- AWS
- PostgreSQL
- Timestamp Authority services
- External networks
- External certification providers

PB-017 does not claim:

- regulatory certification
- external certification
- legal compliance certification
- production readiness

## Required Inputs

PB-017 requires:

- `pb016_governance_improvement_plan.json`
- `pb016_governance_roadmap.json`
- `pb016_governance_action_register.json`

Missing PB-016 evidence is a fail-closed tracking condition.

## Fail-Closed Conditions

PB-017 returns `Decision: BLOCKED` when:

- improvement plan missing
- action register missing
- roadmap missing
- invalid action status
- invalid due date
- overdue critical action detected
- unsupported governance artifact detected
- required PB-016 input schema invalid

## Execution

Run action tracking:

```bash
python3 scripts/pb017_governance_action_tracking.py \
  governance/evidence/pb016_improvement \
  governance/evidence/pb017_action_tracking
```

Expected verified output:

```text
Decision: VERIFIED
PB017_GOVERNANCE_ACTION_TRACKING_VERIFIED
```

## Governance Rule

Evidence before claims.

PB-017 tracks governance execution status only. Open, blocked, missing, invalid, or overdue critical governance actions must remain visible and fail closed where required.
