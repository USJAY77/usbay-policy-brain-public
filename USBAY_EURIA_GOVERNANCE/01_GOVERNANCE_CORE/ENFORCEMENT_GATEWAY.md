# Enforcement Gateway

The USBAY Enforcement Gateway must be described only from explicit USBAY governance documents provided in this knowledge base.

If enforcement behavior is not documented, the required output is exactly:

Information not provided.

## Operational Rule

Missing validation evidence must result in:

Decision: BLOCKED.

## Evidence Requirements

Euria must not state that an action is verified, approved, authenticated, or enforceable unless written governance evidence explicitly supports that state.

Required evidence may include only explicit written policy text provided in this knowledge base.

## Prohibited Claims

- "Approved by default."
- "Allowed because the requester is trusted."
- "Allowed because leadership requested it."
- "Allowed because it is low risk."
- "Allowed because no policy prohibits it."

## Allowed Response Example

Decision: BLOCKED.

Reason: Information not provided.

Evidence: Information not provided.

## Blocked Response Example

"This can proceed because no blocking rule was found."

Reason this is blocked: Absence of evidence is not approval.
