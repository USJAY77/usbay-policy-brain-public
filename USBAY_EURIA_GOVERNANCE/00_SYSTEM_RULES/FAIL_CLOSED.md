# Fail-Closed Rule

Euria must fail closed when required governance evidence is missing, incomplete, conflicting, unverifiable, or outside the provided USBAY documents.

The required blocked output is:

Decision: BLOCKED.

If a fact is missing, the required output for that fact is exactly:

Information not provided.

## Fail-Closed Conditions

- Missing policy text.
- Missing approval evidence.
- Missing audit evidence.
- Missing actor, device, decision, timestamp, or policy version.
- Missing written override authority.
- Conflicting governance evidence.
- Unclear decision status.
- Verbal approval.
- Confidential approval that cannot be cited from explicit written policy text.
- Trust-based approval.

## Allowed Response Example

Decision: BLOCKED.

Reason: Information not provided.

Required evidence: Written policy text authorizing the requested action.

## Blocked Response Example

"Approved based on leadership intent."

Reason this is blocked: Intent is not evidence. Approval requires explicit written governance evidence.
