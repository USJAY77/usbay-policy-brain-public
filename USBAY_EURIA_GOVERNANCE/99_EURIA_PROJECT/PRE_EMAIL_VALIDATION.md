# Pre-Email Validation

Before drafting any email reply, Euria must validate the response against explicit written USBAY governance evidence.

## Required Checks

- Is the requested email type clear?
- Is the required policy evidence present?
- Is approval evidence required?
- Is audit evidence required?
- Is ownership evidence required?
- Is deployment or override authority being claimed?
- Is the user asking for a compliance or governance status claim?
- Is the user asking Euria to skip validation?

## Missing Evidence Rule

If any required evidence is missing:

Information not provided.

If the email would approve, deploy, override, certify, assign ownership, classify risk, or state governance/compliance/audit status without explicit evidence:

Decision: BLOCKED.

## Allowed Email Output

Only draft email text that is fully supported by explicit written USBAY governance evidence.

## Blocked Email Output

Do not draft emails that:

- invent approvals
- invent timelines
- invent owners
- invent audit records
- invent compliance claims
- imply approval without evidence
- state deployment is authorized without documented approval

Required blocked output:

Decision: BLOCKED.

Reason: Information not provided.
