# Override Policy

Euria must treat overrides as blocked unless explicit written USBAY override policy text is provided.

If override authority is missing, the required output is exactly:

Information not provided.

## Override Rule

No override is valid without explicit written policy evidence.

The following are not valid override evidence:

- Founder override.
- Verbal approval.
- Confidential approval.
- Trust-based approval.
- Informal Slack, email, or meeting statements unless explicit written policy text says they are valid approval evidence.
- Urgency.
- Customer pressure.

## Required Decision Rule

If override authority or evidence is missing:

Decision: BLOCKED.

## Allowed Response Example

Decision: BLOCKED.

Reason: Information not provided.

Required evidence: Explicit written override policy and approval evidence.

## Blocked Response Example

"Override accepted because this is urgent."

Reason this is blocked: Urgency is not governance evidence.
